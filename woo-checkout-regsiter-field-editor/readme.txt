### 🎯 Conditional Logic
Conditional Logic allows you to dynamically control the visibility of fields and sections on the checkout page based on specific conditions. This feature works based on cart data, user roles, products, and checkout field values, helping you create a more personalized and streamlined checkout experience.

You can apply conditions based on the following:

**Based on Cart, User Roles, and Products**

Fields or sections will be displayed only if all defined conditions are met. You can set rules such as:

* Cart contains
* Cart does not contain
* Cart only contains
* Cart subtotal equals to
* Cart subtotal greater than
* Cart subtotal less than
* Cart total equals to
* Cart total greater than
* Cart total less than
* User role equals to

**Based on Checkout Fields**

You can also apply conditions based on the values entered or selected in other checkout fields. These include:

* Is empty
* Is not empty
* Value equals to
* Value not equals to
* Value greater than
* Value less than
* Date equals to
* Date not equals to
* Date after
* Date before
* Day equals to
* Day not equals to
* Is checked
* Is not checked

== Frequently Asked Questions ==

= How can I add new text input to the WooCommerce checkout form =
You can add a new text input field to the **WooCommerce checkout form** either by using our **Checkout Field Editor plugin** or by adding custom PHP code to your theme’s functions.php file.

= Is this plugin compatible with WooCommerce Checkout Blocks? =
Yes! The free version of our WooCommerce checkout field editor supports **Text,** **Select,** **Checkbox,** and **Radio Button** fields in **Block Checkout,** while the **Classic Checkout** version includes up to **20 field types**.

With the **PRO version**, you unlock advanced customization features that work in both **Block Checkout and Classic Checkout,** giving you full flexibility to edit, enhance, and optimize your checkout experience.

= Why should I add an additional field to the WooCommerce checkout? =
Adding an additional field to the WooCommerce checkout helps you collect extra information from customers, such as delivery instructions, custom requests, or business details. This improves order processing and allows you to customize the checkout experience based on your needs.

= Why Are My Custom Checkout Fields Not Showing on the WooCommerce Checkout Page? =
If custom checkout fields are not appearing, it depends on whether your checkout uses shortcodes or blocks:

* If the checkout page is **created using shortcode,** the fields should be added and updated in the Classic Checkout Fields section.
* If your checkout is **configured using WooCommerce Blocks,** manage in the fields should be added and updated in the Blocks Checkout Fields section.

= Can I reset WooCommerce checkout fields to default? =
Yes. We’ve added a **Reset to Default** button that lets you instantly restore the original WooCommerce checkout fields. With one click, it removes all custom changes and brings your checkout layout back to the default WooCommerce setup.

= Can I add prices to checkout fields using this plugin? =
Prices are not supported in the free version. To apply prices to checkout fields, you’ll need the [PRO Version](https://jcodex.com/plugins/woocommerce-custom-checkout-field-editor/), which supports multiple field-based pricing types for advanced checkout customization.

= Can I display custom checkout fields on the My Account page? =
In the free version, custom fields can only be displayed and edited on the **Checkout page**. 
The **Pro version** allows custom fields to be displayed and edited on:

* Checkout page

* My Account page

* User profile in the WordPress admin

Field values entered by customers are saved and reused for future orders, so users don’t need to re-enter the same information each time.

= How do I add a text field to the WooCommerce checkout page? =
To add a custom text field to the WooCommerce checkout,
Go To:

* **WooCommerce <span aria-hidden="true" class="wp-exclude-emoji"><span aria-hidden="true" class="wp-exclude-emoji"><span aria-hidden="true" class="wp-exclude-emoji"><span aria-hidden="true" class="wp-exclude-emoji">→</span></span></span></span> Checkout Form Editor**  in your WordPress dashboard.
* Click **Add Field**
* Select **Text** as the field type
* And **save & close** your changes

= How do I make a Select field have no default option? =
* Ensure the first option has a **blank Option Value** and **blank Option Text** so no value is preselected. 
* If needed, add guidance using the **Placeholder** field (for example, **“Select an option”**).

= Will this work with any theme? =
Yes, with most well-coded themes. If a theme hard-overrides checkout templates, keep templates updated and avoid conflicting customizations.

= How do I translate field labels and messages? =
You can translate all field labels and messages using plugins like **WPML,** **Polylang,** or **Loco** Translate. All strings in the plugin are translation-ready, making them easy to localize.

= Does it support validation? =
Yes, the plugin supports built-in validation for common fields such as **Email** and **Phone**. 

= Any performance tips? =
Exclude checkout from **JS/CSS minify/deferral** and from **full-page caching**. Don’t lazy-load critical checkout scripts/styles.

= Is multisite supported? =
Yes, the plugin fully supports WordPress multisite. You can activate it network-wide or activate per site.

= Will this plugin slow down my site? =
No. It is lightweight and optimized for performance. 

= Can I add fields conditionally (only for certain products, roles, or payment methods)? =
Free: manual show/hide only.  
PRO: **Conditional Logic** by **role, country, cart totals, shipping/payment method, product/category, other field values**, etc.

= Can I override address formats and include my custom fields in addresses and emails? =
Free: default formats.  
PRO: **Yes**, override address formats in **My Account, Thank You, and transactional emails** (including your custom meta).
